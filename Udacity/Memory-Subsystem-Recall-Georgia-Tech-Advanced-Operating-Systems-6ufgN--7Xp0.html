recall that in any modern operating system each process is in its own protection domain and usually a separate Hardware address space and the operating system maintains a page table on behalf of each of these processes the page table is the operating systems data structure that holds the mapping between the virtual page numbers and the physical pages where those virtual pages are contained in the main memory of the hardware the physical memory of course is contiguous starting from 0 to whatever is the maximum limit of the hardware capabilities are but the virtual address space of a given process of course is not contiguous in physical memory but it is scattered all over the physical memory and that's in some sense the advantage that you get with page based memory management that a processes notion of its virtual address space being contiguous is not necessarily reflected in the physical mapping of those virtual pages to the physical pages in the main memory
