Network First, then cache

2016-08-08
Network first tries to handle the requests by fetching from the network if the network request fails or times out it will try to fulfill the request from the cache this is the strategy to use for basic read through caching it's also good to use for API requests where you always want the freshest data when it's available but would rather have stale data than no data at all this method does have its flaws though if the user has an intermittent or slow connection they'll have to wait for the network to fail before they'll get the perfectly acceptable content from the cache this can take an extremely long time and is a frustrating user experience network first is ideal for content that is updated frequently and not part of the app shell for example article content social media timelines and game leaderboards
