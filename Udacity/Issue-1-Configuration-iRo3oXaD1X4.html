Issue 1: Configuration

2015-02-23
the first issue is configuration it may not be apparent from the discussion so far that components are typically configurable what this means is that the component vendor provides a means such as a configuration file by which the client using a component can tailor it to a particular situation this gives to the designer a powerful means from managing design trade-offs for example space versus time this flexibility that configuration gives comes at a cost however because configuration is a form of late binding it becomes difficult to unit test the components in the actual usage environments it is also more expensive to document and to deploy them
