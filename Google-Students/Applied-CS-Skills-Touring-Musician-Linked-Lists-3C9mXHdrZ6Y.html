the address of the next node in the list by connecting many nodes together you get a chain or linked list the first node that is the one that doesn't have any notes pointing to it is called the head of the list you may have noticed in this example it is easy to go forward through the list but there is no way to go back this is because this example is a singly linked list if we added an extra field to each node that stored the address of its previous node we would be able to go back as well as forward this would then be a doubly linked list in both of these examples we need to use null fields to represent when there is nothing to link to if we connected the last node in the list to the first head node by storing the address of the head node in the next field of the last node then all nodes would be connected in a loop this is known as a circular linked list as with all data types their advantages disadvantages and trade-offs in using linked lists for example storing a list of
