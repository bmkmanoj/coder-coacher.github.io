Oracle Developer Cloud Service: Synchronizing a Project Using Oracle Enterprise Pack for Eclipse

2015-03-16
each Oracle developer cloud service project is provisioned with maven and git repositories an issue tracker and an instance of a Hudson build server access to these services is available directly from Oracle Enterprise pack for eclipse when you connect the IDE to your cloud project from the cloud view of the IDE you can open the Oracle cloud connection dialog box to connect to your Oracle cloud project until the connection details that you received when you signed up for the service after you connect Oracle cloud expand the tree in the Oracle cloud view to navigate to your project resources and services activate the project and repository in the cloud view to create a local clone of the remote repository after the clone is created the details of the project repository are visible under the source node and the contents under the working directory node right now my local repository is empty because the remote repository was empty if you have or are creating an Oracle Enterprise pack for Eclipse project you can synchronize the project with the Oracle developer cloud service project after the projects are synchronized you can push changes to the remote repository and work with project issues and build jobs from the IDE to synchronize the projects and add the sources to the local repository drag the project from the project Explorer and drop it onto the repository node in the Oracle cloud view when you synchronize your project you can specify if you want maven build files generated or a Hudson build job created after you enter the details and finish the synchronized wizard the IDE pushes the project sources from the local repository to the repository on Oracle developer cloud service project activities are automatically displayed on the home page of your Oracle developer cloud service project for more information see the Oracle developer cloud service documentation page
