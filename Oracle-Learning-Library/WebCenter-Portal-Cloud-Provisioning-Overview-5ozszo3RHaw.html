WebCenter Portal Cloud Provisioning Overview

2017-05-29
Oracle WebCenter Portal Cloud is a subscription service that’s available with Oracle Java Cloud Service Before you provision WebCenter Portal Cloud you’ll need subscriptions to: Oracle Database Cloud Service Oracle Java Cloud Service Enterprise Edition and Oracle WebCenter Portal Cloud WebCenter Portal can be provisioned in two different ways: Compact Topology or Enterprise Topology Compact Topology is recommended for development and testing requirements whereas Enterprise topology is used for production and UAT workloads In this example, let’s configure WebCenter Portal using Compact Topology For this, you’ll need to: Create an Oracle Database Cloud Service instance Create a Java Cloud Service instance and install and deploy WebCenter Portal Let’s start with creating the Oracle Database Cloud Service instance Provide a name for the database instance Make a note of the database service name and password in order to create your Java Cloud Service instance later on and configure the SSH keys to administer your instances Storage Cloud Container creates a back up of the data in your database instance You’ll need to apply patches to the database for Secure Enterprise Search work Connect to the database instance you just created using the private key file and apply patches to the database Now you are ready to create the Java Cloud Service instance Just like the database service, name the Java Cloud Service instance and specify WebCenter Portal for the fusion middleware component to be deployed Then, select the Compute Shape option based on workload requirements Provide the name of the same public key file that was created earlier Enter the database instance credentials to the new weblogic domain created earlier and enable access to Weblogic Administration consoles Distribute the workload across servers in the cluster using a load balancer and store all the data from the Java Cloud Service instance using storage cloud container Your Java Cloud Service instance is now created The default storage space provided in the standard Java Cloud Server instance is not big enough to hold the installation files for WebCenter Portal so you’ll need to scale up the weblogic resource Oracle recommends you create one storage volume of 20Gigabytes for installer software and scripts Create an additional volume to store your content data Your final step is to install WebCenter Portal using a deployment script Connect to the managed server using the SSH and private key to start the deployment then navigate to autodeploy directory and run the wcpautodeploy command The deployment takes about an hour to complete All the messages are stored in the autodeploy log file After the deployment is complete you can access Oracle WebCenter Portal using the Content Endpoint on the Java Cloud Service instance Congratulations! You’ve just deployed WebCenter Portal on the Java Cloud Servcie! Thank you for watching To learn more, visit us online at cloud dot oracle dot com
