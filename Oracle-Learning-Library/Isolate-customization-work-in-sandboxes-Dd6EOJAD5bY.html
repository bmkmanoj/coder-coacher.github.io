Isolate customization work in sandboxes

2017-06-06
when working with oracle salescloud consider that all customizations that effect other users should be done in a staging environment before you begin these customizations it is important for you to familiarize yourself with oracle salescloud sandboxes sandboxes allow you to isolate your work in a staging environment so that the customizations can be completed and tested before they affect other users any changes that you make inside of our sandbox will not be seen by others until the sandbox is published in this video we'll discuss how to work with the Oracle sales cloud sandbox manager we'll use the sandbox manager to make an existing sandbox active the sandbox manager is accessible from the oracle salescloud interface note that you only have access to the sandbox manager if you have the required privileges to do so click current users name to bring up the settings and actions menu then select manage sandboxes next I'm going to select the sandbox in which to work I'm going to use a designated sandbox that has been allotted to me the desired sandbox and then I click set as active whenever you're using a sandbox you'll see this indicator in the global area along with the name of the sandbox that you are using next I'm going to log out and log back in to clear the cache before making any customization in the newly activated sandbox to exit the sand box select the sandbox name then select exit sandbox after completing and testing the changes in your sandbox follow your company's policies and procedures for publishing sandbox for more information see getting started with Oracle sales cloud customizations in the Oracle cloud library at the following location
