welcome to the Oracle Enterprise Manager Cloud Control 12c demo series in this demonstration I will show how Oracle Enterprise Manager cloud control 12c can help you to ensure that future Oracle Fusion Middleware installations and configurations adhere to best practices and standards in your IT organization Oracle Enterprise Manager accomplishes this by enabling administrators to create gold images of Oracle Fusion Middleware software these gold images can be stored as provisioning profiles in a centralized location or in what Enterprise Manager refers to as the software library these provisioning profiles can then be leveraged as the source for future provisioning or cloning operations being able to provision or clone from a provisioning profile simplifies the installation process and ensures that the Oracle Fusion Middleware software being installed across your data center meets IT standards Oracle Enterprise Manager supports creation of three different types of provisioning profiles the first type is a WebLogic domain based provisioning profile a WebLogic domain based provisioning profile includes the weblogic server software installed in the oracle homes for a domain as well as the configuration of that domain this type of provisioning profile requires a source domain that has already been installed perhaps has already been patched configured and discovered as a managed target in Oracle Enterprise Manager at a very high level this type of provisioning profile takes a copy of the existing domains Oracle homes including binaries and patches as well as its domain configuration and stores all of that in the software library this type of provisioning profile would be leveraged when administrators need to both install weblogic server software as well as configure a new WebLogic domain the second type of provisioning profile is an oracle home based provisioning profile an oracle home based provisioning profile includes just the oracle home in which weblogic server software has been installed this type of provisioning profile requires a source oracle home that has already been installed perhaps has already been patched and discovered as a managed target in Oracle Enterprise Manager at a very high level this type of provisioning profile takes a copy of the existing Oracle home including binaries and applied patches and stores that in the software library this type of provisioning profile would be leveraged when administrators need to install just the weblogic server software but they do not need to configure a new a logic domain the third type of provisioning profile is an installation media based provisioning profile an installation media based provisioning profile enables administrators to perform first time installation of Oracle WebLogic server software and first time configuration of a WebLogic domain this type of provisioning profile requires administrators to have already downloaded Oracle WebLogic server software from Oracle either downloaded from edelivery or the Oracle technology Network at a very high level this type of provisioning profile takes a copy of the installation media used for installation and configuration for instance the weblogic server 12 12 jar file for the weblogic server 12 12 release and stores that archive file in the software library this type of provisioning profile would be leveraged when administrators want to perform first time installation of illogic server in cases where they may not already have an existing domain installed and configured or do not wish to clone
