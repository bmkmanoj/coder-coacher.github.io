each project is provisioned with services and tools that you access directly from pages in the interface these include pages for tracking issues viewing code building projects agile development and project administration use the project picker menu to navigate between the projects in your organization the user menu has links to the user preferences and to the Oracle developer cloud service documentation each project has a home page with an activity speed that displays the most recent activities of the project on the right side of the home page the repositories tab lists the project repositories the graphs and statistics tab displays data about the team's tasks and commits and the team tab displays the user names and roles of the project members the project administrator can add and remove members the list of the files and commits made to the repository is displayed on the code page there are two ways to view the sources in the repository use the files view to browse the directory structure and files you use the commits view to see the history of commit transactions and details about the changes made to individual files use snippets to create a shareable library of small pieces of reusable content such as notes or code fragments the code reviews that you have submitted as well as the request for you to review code are visible on the merge requests page when you are looking at a request use the tabs to view the proposed changes to the code and to read and add comments reviewers can approve or reject the changes the track issues page helps track and manage the tasks defects and features of the project in this page you create an update of Hugh's modify details and add comments use the jobs overview page to manage your project build jobs and create new jobs the project are displayed on the deployments page use the project wiki to record and share information with team members use the agile tab to create custom boards to organize issues and to track
