this is SDF in two minutes or less welcome to creating your first sweet cloud project in this session we will cover project types project setup project components in the IDE and saving your project let's get started to create a sweet cloud project in the IDE go to the file menu select the option new and then sweet cloud project you will be presented with several options we will focus on the SDF project section which consists of two project types account customization and sweet app for this video we will select account customization and then click Next now you can set up your project first enter a project name then choose the sweet script version that the IDE will use for code completion purposes notice that this option does not restrict you from having multiple scripts and different versions of sweet script this is just used for Coe completion when you are done click finish in the project will be created now that a project has been created let's explore its structure if you selected the account customization SDF project type you have an account configuration folder in your project if you selected the Suite app SDF project type you see an installation preferences folder all projects have a file cabinet folder which contains a subfolder named Suite scripts the file cabinet folder holds the definitions of suite scripts and files in general there is also a folder to store the custom object definitions for your project the objects folder stores the XML definitions of custom objects for your project such as custom records custom lists and so on another crucial element of the project structure is the manifest file this file contains the project setup properties think about it as a configuration file for your application we will provide more detail about this file in other videos of this series thanks for watching
