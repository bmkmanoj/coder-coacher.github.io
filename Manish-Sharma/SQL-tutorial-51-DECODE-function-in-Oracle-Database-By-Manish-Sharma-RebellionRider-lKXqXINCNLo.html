DECODE function is an old way to perform IF-THEN-ELSE logic in oracle database. It’s old but still supported by the latest versions of oracle database but it is always recommended to use CASE over DECODE for various reasons such as: 1. CASE expression is easy to read in comparison to DECODE function. 2. Using DECODE you can only perform equality tests while using CASE you can perform various logical tests. This means that in CASE expression you can use logical operators while same is not true with DECODE 3. CASE expression is supported by PL/SQL While DECODE is only supported by SQL 4. CASE falls under ANSI SQL/92 standard where DECODE is Oracle proprietary. Let’s quickly take a look at DECODE function’s syntax DECODE is a function thus followed by DECODE keyword we have parenthesis and within parenthesis we have 4 parameters which are value, search value, result and default value. DECODE function compares the value with search value if they match then DECODE function returns the result otherwise default value is returned.
