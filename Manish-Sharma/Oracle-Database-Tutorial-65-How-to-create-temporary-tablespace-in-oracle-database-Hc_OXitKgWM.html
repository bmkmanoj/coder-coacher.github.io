A temporary tablespace is used for temporary segments, which are created, managed, and dropped by the database as needed. These temporary segments are commonly generated during sorting operations such as ORDER BY, GROUP BY, and CREATE INDEX. They are also generated during other operations such as hash joins or inserts into temporary tables. Similar to the permanent tablespace you can create temporary tablespace either as small file or big file tablespace. So let’s write the ddl and see how to create a temporary tablespace. CREATE TABLESPACE temp_RebellionRider TEMPFILE 'C:\APP\TBSP_DEMO\temp_rebel.dbf'SIZE 100M  EXTENT MANAGEMENT LOCAL UNIFORM SIZE 1M; Ok let me first close this script output ok now Let’s quickly take a look at this create tablespace ddl. Statement starts with CREATE and TABLESPACE both of which are oracle reserved keywords. In between these two keywords we have another keyword which is smallfile, which specifies that we want to create smallfile tablespace. By the way this is an optional keyword if you don’t write this, then oracle is gonna create a small file tablespace by default. Next we have the name of the tablespace which is temp_RebellionRider. This name is user defined so you can give whatever name you want to your tablespace. Then there is a TEMPFILE clause, unlike permanent tablespace where we have datafile clause here we have TEMPFILE. Temporary tablespaces are created using temp files instead of datafiles. Temp files are allocated slightly differently than datafiles. Whereas datafiles are completely allocated and initialized at creation time, the temp files are not always guaranteed to allocate the disk space specified. This means that on some UNIX systems a temp file will not actually allocate disk space until a sorting operation requires it. Although this delayed allocation approach allows rapid file creation, it can cause problems down the road if you have not reserved the space that may be needed at runtime. Here I have only added one temp file to our temporary tablespace which is temp_rebel.dbf the size of which is 100MB. Next is Extent management clause where I set it on Uniform mode with size 1 Megabyte. Unlike permanent tablespace, segment space management clause is not available with temporary tablespace. Right while creating a temporary tablespace we have to tell the oracle engine that we are creating a temporary tablespace and to do the same we have to write a mandatory keyword which is TEMPORARY right before the keyword tablespace.
