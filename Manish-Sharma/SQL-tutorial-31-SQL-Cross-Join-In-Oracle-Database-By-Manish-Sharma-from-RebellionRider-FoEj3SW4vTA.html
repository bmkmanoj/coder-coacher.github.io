Cross Join is also known by the name of Cartesian join because it produces Cartesian product of the tables which are participating in the Join queries. What I mean here is that each row in the first table is paired with all the rows in the second table. Let�s have a look of syntax SELECT column names FROM table 1 CROSS JOIN table 2; In comparison to other Joins cross join has the simplest syntax. In the first line we have our Select statement where you can specify the list of columns from both the participating tables. In the second line we have our tables and join clause which is CROSS JOIN. You can write either Cross Join or just put a comma in between the names of both tables. Yes with the cross join we do not have any ON or USING join condition. But you may, however, specify a WHERE and ORDER BY clause. Note if you are using ORDER BY clause then make sure it must the last statement of your SQL query. Hope the syntax is clear now.
