HTML5 Input Types: Email (3/14)

2014-06-19
the email input type does a couple of things the first is a handy front-end validation that means we don't have to write JavaScript to check this field for a valid email of course you should always add back-end validation but in terms of the front-end this adds a native message when an email address is incorrectly entered note though the email address checking isn't as powerful as you might expect so don't rely on it to accept a perfectly valid email we can also allow validation of multiple email addresses by using the multiple attribute on this input element if we take a look at entering two email addresses here you can see that it will give us clear instruction of which email needs correcting now again there is a user experience benefit to using this on a smaller device or any device with an on-screen keyboard you can see here that the buttons change to accommodate common characters used when entering email addresses and this means a shorter entry time for users thus improving the overall experience
